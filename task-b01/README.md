# Задача B1

## Условие

Переделать текущий batch-скрипт из [task-a02](../task-a02/README.md) в _incremental pipeline_, который:
- Обрабатывает только новые файлы.
- Учитывает _late-arriving_ данные.
- Пересчитывает только затронутые месяцы.
- Поддерживает корректные:
  - `cumulative_revenue`
  - `rolling_avg_revenue`

## Входные данные

Директория `input/`

Каждый день туда добавляется новый файл:
```
orders-2024-02-15.csv
orders-2024-03-01.csv
orders-2024-03-10.csv
```
Каждый файл содержит новые заказы.

## Дополнительное усложнение

Может прийти файл:
```
orders-2024-03-15.csv
```
И внутри него будет заказ с:
```
order_date = 2024-01-20
```
Это _late-arriving_ data.

## Требования

### Нужно хранить state

Например:

`processed_files.json`

или _metadata_ таблицу

или `checkpoint.csv`

```
Это сгенеренные требования с кошмарными предложенными опциями. Гораздо проще и безопаснее перекладывать обработанные файлы в папки со старыми поступлениями.
```

### Пересчитывать только затронутые месяцы

Если новый файл содержит:
- февраль
- март

Ты не должен пересчитывать январь.
```
На самом деле, нужно понимать, что файлы с аналитикой будут каждый раз загружаться, генериться и записываться полностью.
Самая дорогая операция - чтение из и запись в файл, поэтому выделение тех кусков, что надо пересчитывать - это усложнение кода и экономия на спичках, поскольку файлы все равно будут читаться/писаться целиком, и это - самая дорогостоящая операция.
Ergo - для упрощения кода и сохранения нужного performance, аналитика каждый раз будет перегенериться целиком.
```

### Корректно поддерживать cumulative

Если февраль изменился —
март `cumulative` должен пересчитаться.

### Rolling 3 месяца

Если добавили данные в январь — февраль и март тоже могут измениться.
