# Задача A2 — Аналитика выручки в cohort-стиле

## Условие

Даны два _CSV_-файла:

`orders.csv` — транзакционные данные заказов

`customers.csv` — метаданные клиентов

Необходимо написать _Python_-скрипт с использованием `pandas`, который формирует помесячный аналитический отчёт с накопительными и скользящими метриками.

## Функциональные требования

### Загрузка и валидация данных

- Загрузить оба файла
- Проверить наличие обязательных колонок
- Корректно распарсить даты
- Ошибочные строки обрабатывать устойчиво (не падать)

### Очистка данных

#### _Orders_

**Удалить** строки, если:
- `quantity <= 0`
- `unit_price < 0`

**Дополнительно**:
- Привести числовые поля
- Некорректные даты → удалить строки

#### _Customers_

- Если `segment` отсутствует → заполнить `"UNKNOWN"`

### Обогащение данных

Соединить:
```
LEFT JOIN on customer_id
```
**Слева** должны стоять `orders`, **справа** - `customers`, т.к.
- `orders` - **факт**, он **первичен**, анализ данных проводится по нему,
- `customers` - **справочник**, он **вторичен**, если запись отсутствует, то для анализа это не так важно.

Добавить поля:
```
revenue = quantity * unit_price
order_month = YYYY-MM из order_date
```

Если клиент не найден:
```
segment = "UNKNOWN"
```

Если есть записи с одинаковым `order_id`:
- полные дубликаты - удаляем, оставляем только одну запись,
- данные отличаются - падаем с ошибкой.

### Агрегация

Группировка по:
- `order_month`
- `segment`

Вычислить:

| Метрика           | Описание                    |
|-------------------|-----------------------------|
| `orders_count`    | число уникальных `order_id` |
| `customers_count` | число уникальных клиентов   |
| `total_revenue`   | сумма `revenue`             |

### Продвинутые метрики

Для **каждого `segment` отдельно (в порядке месяцев)** вычислить:
- Накопительную выручку
  ```
  cumulative_revenue
  ```
  Это значит, что для каждого месяца в сегменте считается сумма значний `total_revenue` за предыдущие месяцы, включая этот.
  ```
  cumulative_revenue = cumsum(total_revenue)
  ```
- Скользящее среднее выручки (**3 месяца**)
  - Окно = 3
  - `min_periods=1`
  ```
  rolling_avg_revenue
  ```
  Это значит, что для каждого месяца в сегменте считается средняя выручка за этот и два предыдущих месяца. Если данных за два предыдущих месяца нет, то за сколько есть.
  ```
  rolling_avg_revenue = rolling(window=3, min_periods=1).mean()
  ```

### Глобальные итоги (_grand totals_)

Добавить строки:
- `segment = ALL`
Для каждого месяца агрегировать по всем сегментам
(с теми же метриками `cumulative` и `rolling`)

### Выходной результат

Отсортировать:
- `order_month ASC`
- `segment ASC`

Сохранить:
```
monthly_summary.csv
```

## Специально заложенные _edge cases_

- Отсутствующий клиент → `UNKNOWN`
- Пустой сегмент → `UNKNOWN`
- Некорректная дата
- Отрицательная цена
- Нулевая `quantity`
- Отсутствие сегмента в месяце (проверка `rolling`)
- Взаимодействие `cumulative` и `rolling` метрик
